# User Interface Design Document

## 1. UX/UI Philosophy
The Integration-445 Frontend is engineered as a Single Page Application (SPA) utilizing React.js. The design philosophy centers on **Data Visibility, Administrative Efficiency, and Strict Role-Based Context**. The interface must adapt dynamically so that users only see components relevant to their authorization level, minimizing cognitive load and preventing unauthorized actions.

## 2. Global Layout & Navigation

### 2.1 Context-Aware Sidebar Navigation
The application employs a persistent sidebar. Its rendering logic is tied directly to the JWT `role` claim.
- **Admin**: Views all modules (Dashboard, Employees, Payroll, Attendance, Dividends, Reports, System Users, Audit Logs).
- **HR**: Views Dashboard, Employees, HR Reports, Profile.
- **Payroll**: Views Dashboard, Payroll, Attendance, Payroll Reports, Profile.
- **Employee**: Views restricted Self-Service modules (My Payroll, My Attendance, My Profile).

### 2.2 Top Navigation Bar
- **Breadcrumbs:** Contextual indicators of the current routing path (e.g., `Home / Employees / Detail`).
- **Profile Menu:** Quick access to User Profile settings and the Secure Logout action.

## 3. Screen Specifications & Interactions

### 3.1 Authentication Module
- **Login Screen:** A clean, centered modal. Inputs for Username/Email and Password. Includes client-side validation (disabling submit button if empty). On failure, displays explicit error messages from the backend (e.g., "Account locked").
- **Password Recovery Wizard:** 
  - *Step 1:* Email input -> Triggers backend SMTP service.
  - *Step 2:* 6-digit OTP input -> Features auto-advancing single-character input boxes for UX. Displays a 10-minute countdown timer.
  - *Step 3:* New password entry with strength indicator.

### 3.2 The Executive Dashboard
Designed to provide a high-level operational overview at a single glance.
- **Top Ribbon (KPI Cards):**
  1. *Total Employees:* Sourced from SQL Server.
  2. *Total Monthly Payroll ($):* Sourced from MySQL.
  3. *Attendance Rate (%):* Calculated cross-database metric.
- **Main Chart Area:** A dual-axis line/bar chart displaying "Revenue vs Expenses" over a 6-month trend, powered by libraries like `Recharts` or `Chart.js`.
- **Alerts Panel (Right-Hand Side):** A vertically scrolling feed of critical business alerts (Birthdays, Work Anniversaries, >3 Leave Days, >20% Salary Anomalies). Items are color-coded by severity (Red = Critical, Yellow = Warning, Blue = Info).

### 3.3 HR Management (Employees Screen)
- **Data Grid:** A comprehensive table displaying Employee ID, Name, Department, Position, and Status.
- **Interactions:**
  - *Filtering:* Dropdowns to filter by Department or Status (Active, Probation, On Leave, Inactive).
  - *Search:* Debounced search bar querying Full Name.
  - *Actions Column:* Edit (Pencil icon) and Deactivate (Trash icon).
- **Creation/Edit Modal:** A slide-out drawer or central modal capturing all required HR fields. Validates date formats and prevents submission of negative numerical values.

### 3.4 Payroll & Attendance Processing
- **Payroll Table:** Displays financial data. For authorized roles (Admin, Payroll), the `Base Salary`, `Bonus`, and `Deductions` fields are inline-editable. Upon pressing 'Enter' or losing focus, a `PUT` request is silently dispatched, and `Net Salary` is recalculated and updated dynamically on the UI without a page refresh.
- **Attendance Summaries:** Visual indicators (progress bars or heatmaps) showing Leave/Absent ratios per employee to quickly identify absenteeism trends.

### 3.5 System Administration
- **Audit Logs View:** A strict, read-only data table chronologically detailing system events. Designed for compliance auditing, featuring timestamp sorting and action filtering (e.g., filter by `PASSWORD_CHANGED`).
